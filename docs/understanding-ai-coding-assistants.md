## The Three Forms of Claude (5-Minute Overview)

Confused about "Claude Code" vs "Claude Desktop" vs "Claude"? You're not alone. Let's clarify:

### Claude in the Browser (chat.anthropic.com)
- **What:** Web-based chat interface
- **Best for:** Questions, research, writing, one-off tasks
- **Limitation:** Can't access your local files or modify code projects
- **Who uses it:** Everyone

### Claude Desktop
- **What:** Desktop application (Windows/Mac)
- **Best for:** Document workflows, persistent conversations, local file access
- **Limitation:** Not designed for software development
- **Who uses it:** Professionals, writers, business users

### Claude Code (Command Line)
- **What:** Terminal-based AI coding assistant
- **Best for:** Software development, multi-file projects, developer workflows
- **Key feature:** Reads your entire codebase, suggests multi-file changes
- **Who uses it:** Developers, DevOps, data scientists, technical writers

**This course focuses on Claude Code** - the professional developer tool.

## The Industry Shift: From Autocomplete to Agentic AI (Early 2025)

**Timeline:**
- **2021**: GitHub Copilot (autocomplete while you type)
- **2023**: ChatGPT Code Interpreter (run code in chat)
- **2024**: Claude Code, Cursor, Aider (project-aware agents)
- **2025**: Multi-agent systems becoming standard

**Key trend:** AI assistants are now **"agentic"** - they:
- Understand your entire project context
- Perform multi-step tasks autonomously
- Integrate with your development workflow (Git, Docker, etc.)
- Work alongside you, not just suggest next lines

**Why this matters:**
- **For developers:** 10x productivity boost
- **For non-technical professionals:** Essential literacy for working with developers
- **For everyone:** AI skills are converging across disciplines

### The CLI Advantage for Knowledge Work

**Why terminal-based AI beats web UIs:**

1. **Focus** - No distracting UI elements, just you and the AI
2. **Speed** - Type commands faster than clicking through menus
3. **Context** - Works directly in your project directory
4. **Integration** - Combine with Git, Docker, other CLI tools
5. **Automation** - Script repetitive tasks
6. **Portability** - Same interface everywhere (Windows/Mac/Linux)

**Example:** Writing a PRD

**SaaS approach:**
1. Open PRD template tool
2. Click through sections
3. Copy/paste to Claude web
4. Copy back to tool
5. Format in the tool
6. Share via tool's system

**CLI approach:**
```bash
cd my-project
claude "Create a PRD for the user authentication feature following our template"
# AI generates comprehensive PRD in markdown
# Edit with Ctrl+G in your text editor
# Commit to Git
# Share via any method you want
```

**Result:** Faster, more flexible, directly integrated with your workflow.
